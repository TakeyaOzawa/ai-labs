# gws-trend-extractor 中間ファイル出力フォーマット

## Drive APIクエリの取得方針

各種別のファイル取得では、以下の条件で検索する:

**取得条件（OR）:**
- `modifiedTime` が対象日の範囲内（開始〜終了）
- **または** `createdTime` が対象日の範囲内（開始〜終了）

これにより、対象日に「更新された」ファイルだけでなく「新規作成された」ファイルも漏れなく取得できる。

**クエリテンプレート:**
```
(modifiedTime > "{開始UTC}" and modifiedTime < "{終了UTC}") or (createdTime > "{開始UTC}" and createdTime < "{終了UTC}")
```

加えて共通条件:
- `trashed = false`（ゴミ箱除外）
- MIME type フィルタ（種別ごと）

## ミーティング議事録の特別処理

Google Docsの中には、Google Meetの文字起こしやGeminiによる要約が自動生成されたミーティング議事録が含まれる。これらは通常のドキュメントとは別に特別扱いする。

**識別方法:**
- ファイル名に「議事録」「Meeting notes」「文字起こし」「Transcript」を含む
- オーナーが「Google Meet」または自動生成系のサービスアカウント
- ファイル名が日付+会議名のパターン（例: `2026-05-04 週次定例`）

**議事録から抽出すべき情報:**
1. **会議名**: ファイル名から抽出
2. **参加者**: ドキュメント内の参加者リスト
3. **日時**: ファイル名またはドキュメント内の日時情報
4. **議題**: 主要な議題を箇条書きで列挙
5. **結論・決定事項**: 会議で決まったことを明確に記載
6. **ネクストアクション**: 誰が・何を・いつまでに行うかを明記
7. **未決事項**: 結論が出なかった議題があれば記載

**出力フォーマット（議事録セクション）:**
```markdown
### 🗓️ ミーティング議事録

#### {会議名}（{日時}）
- **参加者**: {参加者リスト}
- **議題**: {主要議題}
- **結論・決定事項**:
  - {決定事項1}
  - {決定事項2}
- **ネクストアクション**:
  - [ ] {担当者}: {アクション内容}（期限: {日付}）
- **未決事項**: {あれば記載}
```

## 概要の記載ルール

**概要は「何のドキュメントか」だけでなく「具体的に何が書かれているか」まで踏み込んで記載すること。**

### NG例（浅い概要）

> OACのリテンション施策に関する提案資料。既存顧客の継続率向上を目的とした戦略的な取り組み。

### OK例（深掘りした概要）

> OACのリテンション施策に関する提案資料。具体的には、(1) 契約満了6ヶ月前からの早期アプローチ施策、(2) 満了時の乗り換え・再リース提案フロー、(3) 離脱理由分析に基づくセグメント別リテンション施策の3本柱で構成。現状のリテンション率は約XX%で、施策実施により+5pt改善を目標としている。

### 概要に含めるべき要素

- **戦略・方針書の場合**: 施策の具体的内容（3本柱等）、数値目標、現状の課題、関連ドキュメントとの関係
- **要件定義書の場合**: 対象機能の概要、主要な技術要件、スコープ、制約事項
- **KPI・データシートの場合**: 管理している主要指標、直近のトレンド、前期比較
- **研修・教育資料の場合**: 対象者、主要なトピック、実施予定
- **業務フロー・マニュアルの場合**: 主要ステップの概要（Step1→Step2→...）、対象業務、前提条件

## 修正内容の記載ルール

- **新規作成の場合**: 「新規作成」と明記。ドキュメントの主要な構成・内容を記載
- **更新の場合**: 追加されたセクション・変更された数値・削除された内容・修正された誤りを特定
- **推定の場合**: 「（推定）」を付記
- **特定できない場合**: 「修正内容の詳細は不明（ドキュメントを直接確認してください）」と記載

## ネクストアクションの記載ルール

| 種類 | 例 |
|---|---|
| **確認・キャッチアップ** | 「全メンバーが施策の方向性を確認すること」 |
| **承認・意思決定** | 「5/10の経営会議で承認予定」 |
| **レビュー・フィードバック** | 「関係者からのフィードバックを5/15までに収集」 |
| **実装・開発** | 「要件定義完了後、設計フェーズに移行予定」 |
| **周知・展開** | 「次回全体定例で施策を共有予定」 |
| **データ更新** | 「月次KPIの5月分データを月末に追加予定」 |
| **不明** | 「ドキュメント内に明示なし。オーナーに確認推奨」 |
