# GitHub Repo Analysis レポート統合フォーマット

markdown-reporter が github-repo-analysis パイプラインの一時ファイルを統合する際のフォーマット指示。

## 入力ファイル

| ファイル | 内容 | 必須 |
|----------|------|------|
| `.tmp_{slug}_github.md` | GitHub API調査結果 | ✅ |
| `.tmp_{slug}_refs.md` | 参照先リポジトリ調査 | ❌（スキップ可） |
| `.tmp_{slug}_web.md` | Web調査結果 | ❌（スキップ可） |
| `.tmp_{slug}_codebase.md` | コードベース分析結果 | ❌（スキップ可） |

## 出力フォーマット

### フロントマター

```yaml
---
date: {YYYY-MM-DD}
repository: {owner}/{repo}
slug: {slug}
url: https://github.com/{owner}/{repo}
stars: {N}
language: {primary_language}
license: {license}
analyzed_by: github-repo-analysis-pipeline
---
```

フロントマターの値は `.tmp_{slug}_github.md` のフロントマターから取得する。

## 統合ルール

1. **重複排除**: 同じ情報が複数ファイルに含まれる場合、最も詳細なものを採用
2. **セクション対応**: 入力ファイルが存在しない場合、該当セクションは「（調査未実施）」と記載
3. **情報の再構成**: 単純な結合ではなく、読みやすい流れに再構成する
4. **出典明示**: 各セクションの情報がどの入力ファイルに由来するかは明示不要（統合レポートとして一体化する）
5. **フロントマターの値**: github.mdのフロントマターから取得。存在しない場合は本文から推定
