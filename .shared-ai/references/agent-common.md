# エージェント共通規約

各エージェントプロンプトから該当セクション番号を指定して参照する。

## §1 基準日付の決定

基準日がプロンプトで指定されている場合はそれを使用。指定がなければスクリプトで確定:
- 前日取得（*-trend-scout系）: `python3.12 ~/scripts/get-jst-date.py --yesterday`
- 当日取得（web-searcher等）: `python3.12 ~/scripts/get-jst-date.py`

**AIモデルの推測に頼らず、必ずスクリプトで確定させること。**

## §2 実行モード判定

- **自律実行**: プロンプト内にテーマ/パス/入力が含まれている or 「実行してください」等の指示がある → 確認なしで一気通貫実行
- **対話モード**: 上記に該当しない → 調査方針をユーザーに提示し確認後に実行

## §3 ファイル書き込み戦略

`readFile: ~/.shared-ai/references/file-write-strategy-guide.md` の手順に従うこと。

## §4 完了報告フォーマット

**完了時は以下の形式のみで報告すること。レポート全文やファイル内容は絶対に返さないこと:**

```
✅ {エージェント名} 完了
- 出力: {ファイルパス}
- {メトリクス}: {値}
```

メトリクス例: 件数、メッセージ数、スレッド数、テーマ数、ソース数

## §5 Web検索系共通規約

### 重複排除（Phase 0）

過去3日分のレポートからURLを抽出し「既出URLリスト」を作成する。
ただし以下は既出リストに含めない: パスが `/` で終わるURL、`/archive/`・`/feed/`・`/news/`・`/blog/`・`/research/`・`/latest` で終わる一覧ページURL。

### フィルタリングルール

各検索結果に適用:
- publishedDateが対象日±1日の範囲外 → 除外（null/未提供の場合は通す）
- 既出URLリストに含まれる → 除外
- トップページ（パスが `/` のみ、または上記一覧パターンで終わる）→ 除外

### コンテキスト節約

- 検索回数上限はプロンプトで指定された値に従う（デフォルト: 15回）
- **1回検索するごとに即座にfsAppendで一時ファイルに書き出す**
- 検索結果をコンテキストに保持し続けない

## §6 サブエージェント委譲規約

1. contextFiles にサブエージェントのプロンプトファイルを指定する
2. prompt に必要なパラメータ（対象日、入出力パス等）を全て含める
3. **完了報告のみ返すよう指示する**（§4 フォーマット。レポート全文を返さない）
4. サブエージェントは1つずつ順番に実行する（並列実行禁止）
5. エラー時はスキップして次に進む
