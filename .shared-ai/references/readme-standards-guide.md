# README 準拠規格・設計根拠ガイド

README記述ルール（`rules/quality/readme-guide.md`）の設計根拠と準拠規格の詳細。

## IEEE 1063:2001 からの採用要素

| IEEE 1063 セクション | README での対応 | 採用理由 |
|---|---|---|
| Identification | §1 概要 | ドキュメントの目的・スコープを明確化 |
| Prerequisites | §2 前提条件 | 利用開始前に必要な環境を明示 |
| Installation | §3 セットアップ | タスク指向のインストール手順 |
| Orientation | §4 アーキテクチャ | システム全体像の把握を支援 |
| Reference | §5 ディレクトリ構成 | 構成要素の一覧と関係性 |
| Procedures | §6 運用手順 | タスク指向の操作ガイド |
| Troubleshooting | §7 トラブルシューティング | 問題解決の自己支援 |
| Appendix | §8〜 | 制約事項、関連ドキュメント、変更履歴 |

## ISO/IEC 26514:2022 からの採用要素

- **タスク指向**: 「何ができるか」ではなく「何をしたいか」を起点に構成する
- **対象読者の明示**: 読者のスキルレベル・目的に応じた情報の粒度を設定する
- **段階的開示**: 概要→詳細の順で情報を提示し、必要な深さまで掘り下げられる構造にする

## ITIL v4 からの採用要素

- **構成アイテム（CI）一覧**: ディレクトリ構成をCI一覧として整理し、各要素の責務を明記する
- **依存関係の可視化**: コンポーネント間の参照関係を図示する
- **変更管理**: 変更履歴セクションで変更の追跡性を確保する
