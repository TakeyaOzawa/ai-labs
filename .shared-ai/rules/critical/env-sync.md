# 環境変数同期ルール

`.zshrc` / `.bashrc` で環境変数の `export` 行が追加・変更・削除された場合、
またはエージェントプロンプト内で新しい環境変数への依存が追加された場合、
以下の3ファイルへの反映が必要か確認すること。

## 同期対象ファイル

1. **`scripts/platform-commands.sh`** — `source-env` セクション内の grep パターン（eval行とenv行の2箇所）
2. **`scripts/setup/check-env.py`** — `REQUIRED_VARS` または `OPTIONAL_VARS` リスト
3. **`scripts/README.md`** — 環境変数テーブルおよび `source-env` 管理変数リスト

## 判断基準

### 同期が必要なケース

- `.zshrc` に新しい `export` 行を追加し、それが `scripts/` 配下のスクリプトまたは MCP 経由で使用される場合
- エージェントプロンプト内で `環境変数 XXX` への依存が記述された場合
- 既存の環境変数名をリネームした場合（旧名の削除 + 新名の追加）

### 同期が不要なケース

- `PATH` や `AWS_PROFILE` など、スクリプト実行に直接関係しない環境変数の変更
- 値の変更のみ（変数名が変わらない場合）
- コメントアウトの追加・削除のみ

## 分類基準

| 分類 | 条件 |
|------|------|
| 必須（REQUIRED_VARS） | 未設定時にスクリプトが `KeyError` や明示的エラーで停止する |
| オプション（OPTIONAL_VARS） | フォールバック値がある、または特定パイプラインでのみ使用 |
