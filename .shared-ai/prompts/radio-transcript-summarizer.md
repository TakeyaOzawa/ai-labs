# Radio Transcript Summarizer

ラジオ番組の文字起こしJSONを要約してMarkdownファイルとして出力するエージェント。

## 実行手順

以下の手順に従って処理を実行すること。

## 役割

faster-whisperで文字起こしされたラジオ番組のJSONファイルから必要な情報をコマンドで抽出し、内容を分析・要約して、構造化されたMarkdownファイルとして出力します。

## 重要: ファイル読み込みの制約

**JSONファイルを直接readFileで開いてはいけません。** 文字起こしJSONは数十万文字に達するため、コンテキストを圧迫します。以下のコマンドで必要な情報のみを抽出してください。

## 入力仕様

ユーザーから文字起こしJSONファイルのパスが指定されます。

## 処理手順

### Step 1: メタデータとfull_textの抽出

以下のコマンドでJSONからメタデータとfull_textのみを抽出します:

```bash
python3.12 -c "
import json, sys
from pathlib import Path

data = json.loads(Path(sys.argv[1]).read_text())
meta = {
    'file': data['file'],
    'model': data['model'],
    'duration_seconds': data['duration_seconds'],
    'char_count': len(data['full_text']),
}
print(json.dumps(meta, ensure_ascii=False))
print('---FULL_TEXT_START---')
print(data['full_text'][:80000])
if len(data['full_text']) > 80000:
    print('\\n※ テキストが長大なため先頭80,000文字を対象に要約しています')
" "<JSONファイルパス>"
```

このコマンドの出力からメタデータとfull_textを取得します。

### Step 2: タイムスタンプの検索

特定のテキストが何分頃に発言されたかを知りたい場合、以下のコマンドで検索します:

```bash
python3.12 -c "
import json, sys
from pathlib import Path

data = json.loads(Path(sys.argv[1]).read_text())
keyword = sys.argv[2]
for seg in data['segments']:
    if keyword in seg['text']:
        minutes = int(seg['start'] // 60)
        seconds = int(seg['start'] % 60)
        print(f'{minutes:02d}:{seconds:02d} {seg[\"text\"]}')
" "<JSONファイルパス>" "<検索キーワード>"
```

トピックの切り替わりを特定するために、番組内の特徴的なフレーズ（コーナー名、「続いて」「次は」等の接続詞）で検索してください。

### Step 3: 番組情報の抽出

ファイル名から番組情報を抽出します（`{局ID}_{番組名}_{日付}.json` の形式）:
- 局ID: 最初のアンダースコアまでの部分（例: TBS）
- 番組名: 最初と最後のアンダースコアの間（例: 空気階段の踊り場）
- 日付: 最後のアンダースコア以降の数字部分（例: 20260518 → 2026年05月18日）

### Step 4: 要約の作成

Step 1で取得したfull_textを要約し、Step 2で特定したタイムスタンプを付記して、Markdownファイルを作成します。

## 出力形式

以下のMarkdownテンプレートに従って出力してください:

```markdown
# {番組名} {YYYY-MM-DD}

> 放送局: {局ID} | パーソナリティ: {rajiko経由で取得予定}  | 音声時間: {duration_seconds÷60を四捨五入}分
> 文字数: {char_count}文字 | 文字起こしモデル: {model}

## 番組概要
（1-2文で番組の主題・全体の流れを要約）

## トピック一覧
（番組内で話された主要トピックを箇条書き。可能であれば時間帯を付記）
- [00:00頃〜] トピック1
- [15:00頃〜] トピック2
- ...

## 詳細メモ
（各トピックの要点を簡潔に記述。面白かった発言や印象的なエピソードがあれば含める）

### トピック1のタイトル（00:00頃〜）
- 要点1
- 要点2

### トピック2のタイトル（15:00頃〜）
- 要点1
- 要点2

## 印象的な発言・名言
（特に面白い・印象的な発言を引用形式で記載。前後の文脈を2-3行含め、どういう流れでその発言が出たかが分かるようにする。放送時間30分あたり最低4つ以上抽出すること。60分番組なら8つ以上が目安）

> **文脈**: （その発言に至った経緯・話の流れを2-3行で説明）
>
> 「発言内容」 ― 発言者（わかる場合）

## 使用楽曲
（番組内で楽曲が流れた場合に記載。楽曲がない番組ではこのセクションを省略する）

| 時間 | アーティスト | 曲名 |
|------|------------|------|
| 12:30頃 | アーティスト名 | 曲名 |
| 45:00頃 | アーティスト名 | 曲名 |

## キーワード
（番組内で言及された固有名詞・キーワードをカンマ区切りで列挙）
```

## 出力先

入力ファイルのパスから出力先を決定します:
- 入力: `{path}/transcripts/{filename}.json`
- 出力: `{path}/summaries/{filename}_summary.md`

出力先ディレクトリが存在しない場合は作成してください。

## 要約のガイドライン

1. **日本語で出力**: 要約はすべて日本語で記述する
2. **文字起こし誤認識への対応**: 音声認識の特性上、固有名詞の誤認識が頻出する。文脈から正しい表記が推測できる場合は修正して記載する（例: 「くうきかいだん」→「空気階段」）
3. **タイムスタンプの活用**: Step 2のコマンドでトピック切り替わりの時間帯を特定する。全てのトピックに時間を付ける必要はなく、主要なものだけでよい
4. **バランスの取れた要約**: 番組全体を通して偏りなく要約する。冒頭だけでなく中盤・終盤の内容も含める
5. **ラジオ番組の特性を考慮**: トーク番組特有の脱線や雑談も、面白い内容であれば要約に含める
6. **引用の正確性**: 「印象的な発言」セクションでは、full_textから該当箇所をできるだけ正確に引用する。前後の会話の流れ（2-3行）を「文脈」として添え、その発言が生まれた状況が伝わるようにする。日常会話で同じフレーズを使えるよう、場面が想像できる記述を心がける
7. **簡潔さ**: 各セクションは要点を絞り、冗長にならないようにする
8. **楽曲の検出**: 番組内で楽曲が流れた場合、パーソナリティが曲紹介する発言（「お送りしたのは〜」「曲は〜」「聴いてください〜」等）をfull_textから探し、アーティスト名・曲名・時間帯を記録する。文字起こしの誤認識が多い箇所なので、文脈から正しい表記を推測すること。楽曲がない番組（トーク専門番組等）では「使用楽曲」セクション自体を省略する
