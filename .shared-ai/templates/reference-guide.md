# Reference Guide テンプレート

`references/` 配下に `-guide` ファイルを新規作成する際のテンプレート。
命名規則は `~/.shared-ai/references/agent-creation-guide.md` の命名規則セクションを参照。

## テンプレート

```markdown
# {ガイドタイトル}

{1〜2文の概要。このガイドが何を定義し、誰が参照するか}

## {セクション1: 設計原則 / 判断基準 / 分類}

<!-- 
  このガイドの中核となるルール・基準を記載。
  テーブル形式を推奨（判断基準、分類、対応表等）。
-->

| {基準} | {条件} | {結果/対応} |
|--------|--------|-------------|
| {値}   | {条件} | {結果}      |

## {セクション2: 手順 / パターン}

<!--
  具体的な手順やパターンを記載。
  コードブロック、番号付きリスト、図を活用。
-->

### {サブセクション}

```
{コード例 / 構成図 / コマンド例}
```

## {セクション3: 実装例 / 現在の実装}

<!--
  現在の実装状況をテーブルで整理（該当する場合）。
  将来の追加時の参考になる。
-->

| {対象} | {実装} | {備考} |
|--------|--------|--------|
| {値}   | {値}   | {値}   |

## チェックリスト（新規追加時）

<!--
  このガイドに基づいて新規追加する際の確認項目（該当する場合）。
-->

- [ ] {確認項目1}
- [ ] {確認項目2}
- [ ] {確認項目3}
```

## 記述ルール

1. **タイトルは「〜ガイド」で終える** — referenceとしての性質を明示
2. **概要は2文以内** — 何のガイドか、誰が参照するかを簡潔に
3. **テーブルを多用する** — 判断基準・分類・対応表はテーブルで整理
4. **コードブロックで具体例を示す** — 抽象的な説明だけでなく実例を含める
5. **チェックリストは任意** — 新規追加手順がある場合のみ
6. **他のreferencesへの参照は相対パスで** — `agent-prompt-guide.md` のように
7. **サイズ目安: 3〜8KB** — 大きくなりすぎたら分割を検討
