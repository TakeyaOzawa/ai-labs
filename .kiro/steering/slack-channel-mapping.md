# Slackチャンネルマッピング

Slackチャンネルの調査・収集を行うエージェント（slack-trend-scout、slack-digest-scout等）が参照するチャンネル情報。

## 調査対象チャンネル

日次・週次のSlack調査で対象とするチャンネル。

| チャンネルID | チャンネル名 | 用途 | 日次収集 |
| --- | --- | --- | --- |
| `C05B4AZ7ZMM` | エンジニア用 | PR共有、設計議論、技術的な相談 | ✅ |
| `C05TJBT6BM2` | エンジニア他部署連絡用 | 要件や方針、運用面の相談 | ✅ |
| `C5L91295J` | 不具合報告 | 他部署からの不具合報告 | ✅ |
| `C02S55ZN0U9` | 作業依頼・質問 | 他部署からの作業依頼・質問・相談 | ✅ |

| --- | --- | --- | --- |

## リアクションの意味マッピング

Slackメッセージのリアクションから対応状況を判定する際の基準。

| カテゴリ | リアクション例 | 意味 |
| --- | --- | --- |
| ✅ 完了 | `済`, `done`, `sumi`, `sumi1`, `済1`, `済2`, `done2` | 対応完了 |
| 👀 確認 | `eyes`, `確認`, `kakunin_shimasu`, `みました_mimashita` | 確認中・認識済み |
| 👍 承認 | `lgtm`, `lgtm2`, `承認`, `承認5` | レビュー承認 |
| 🙏 感謝 | `arigatog`, `arigatougozaimasu`, `ありがとうござ` | 感謝・完了 |
| 👌 了解 | `shochi`, `大丈夫です`, `okemaru_おけまる` | 了承 |
| 🙋 対応中 | `raising_hand_google_hangout`, `woman-raising-hand` | 対応者が名乗り出た |

### 注目度の判定基準

- 3つ以上の異なるリアクション → 高注目
- 合計5人以上のリアクション → 高注目
- リアクションなし + 質問・依頼系メッセージ → 未対応の可能性（要注記）

## メッセージのカテゴリ分類

日次レポートでトピックを分類する際の基準。

| カテゴリ | アイコン | 対象 | 判定基準 |
| --- | --- | --- | --- |
| 障害・緊急対応 | 🔴 | エラー通知、バッチ失敗、本番障害 | bot_messageのエラー通知、「最重要」タグ付き不具合報告 |
| 設計・開発 | 🟡 | PR共有、設計レビュー、方針決定 | GitHub PR/Issueリンク含む、設計プラン共有 |
| 運用・デプロイ | 🟢 | デプロイ通知、バッチ実行、DB操作 | パイプライン通知、手動実行報告 |
| 作業依頼・質問 | 📋 | フォーム経由の依頼、他部署からの相談 | 作業依頼/質問フォーム経由、メンション付き質問 |
| データ移行 | 🔄 | Kintone移行、ETL、データ検証 | 移行手順、ETL実行、データ整合性確認 |
| 未解決・要注意 | ⚠️ | リアクションなし、結論未定 | リアクション・返信なしで放置 |
| 勤怠・雑談 | 💬 | 出退勤、休憩、離席 | 勤怠キーワード、短文メッセージ（レポートからは除外） |
